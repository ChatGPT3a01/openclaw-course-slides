22.1 先來解決點名問題

CH8 教過用 AI 視覺模型看圖——但傳教室照片過去，龍蝦只會說「裡面有很多學生」。問題出在哪？因為 AI 視覺模型是「腦」，負責理解；但它沒有「眼」，看不清楚細節。

🧠 CH8 Vision（AI 的「腦」）

「教室裡有很多學生」

能描述畫面，但數不出人數

👁️ YOLO6（AI 的「眼」）

「person: 28, chair: 35」

精準框出物件，報出數量

比較項目CH8 AI 視覺（腦）YOLO6 物件偵測（眼）
擅長描述、理解、對話框出、計數、定位
點名回答「有一些學生在上課」「person: 28 個」
速度雲端 API（1-3 秒）本地即時（毫秒級）
💡 Vision 負責「理解」，YOLO6 負責「看清楚」——兩者搭配才最強！先用 YOLO6 數人頭，再讓 AI 用自然語言回覆。

完整程式碼（只要 10 行）

from ultralytics import YOLO

# 1. 載入模型（第一次會自動下載）
model = YOLO("yolo11s.pt")

# 2. 偵測圖片
results = model("classroom.jpg")

# 3. 顯示結果
results[0].show()       # 彈出標註過的圖片
results[0].save()       # 儲存標註圖片
print(results[0].boxes) # 印出偵測框座標

偵測結果長這樣

person: 0.95 (280, 120, 380, 450)
chair:  0.88 (100, 300, 200, 500)
laptop: 0.91 (400, 200, 520, 320)
# 類別: 信心分數 (x1, y1, x2, y2) 座標框
✅ 檢查點：看到彩色框框和物件列表就成功了！每個物件都有名稱、信心值和座標。

## 工具名稱
yolo-vision

- 動作: |

    推播到 LINE：「第 X 節：到場 N 人 / 應到 32 人」。
    如果人數低於 25 人，額外提醒老師注意。

效果

每節上課鐘響後 10 分鐘，龍蝦自動拍照、偵測、推播結果到你的 LINE。你只需要看通知就好。

💡 隱私安全——YOLO6 數人頭 ≠ 臉部辨識。它只偵測「這裡有個人形」，不知道是誰，不儲存臉部特徵，不涉及個資法。
⚠️ 需要電腦接 Webcam 並保持開機。搭配 CH21 排程可設定只在上課時間啟動。

22.7 進階：即時偵測

不只分析照片——接上 Webcam，讓龍蝦持續「盯著看」。

即時偵測 Python 程式碼（精簡版）

from ultralytics import YOLO
import cv2

model = YOLO("yolo11s.pt")
cap = cv2.VideoCapture(0)  # 開啟 Webcam

while True:
    ret, frame = cap.read()
    if not ret: break
    results = model(frame, verbose=False)
    annotated = results[0].plot()    # 畫上偵測框
    cv2.imshow("YOLO6 Live", annotated)
    if cv2.waitKey(1) == ord('q'): break

cap.release()
cv2.destroyAllWindows()

常見問題 + 小結

  • Q：需要 GPU 嗎？
    A：不需要！yolo11n 在 CPU 上就能跑，5 年前的筆電也行。GPU 只是跑更快。
  • Q：能偵測什麼東西？
    A：預設 80 種常見物件（人、車、動物、家具、3C 等）。用 model.names 查看完整清單。
  • Q：能和 CH8 Vision 同時用嗎？
    A：最佳搭配！YOLO6 先「偵測 + 計數」，AI 模型再「理解 + 回應」。
  • Q：偵測不準怎麼辦？
    A：換大一號模型（n → s → m）、調高信心閾值、確保光線充足。
  • Q：會侵犯隱私嗎？
    A：YOLO 只偵測「人形輪廓」，不辨識身份、不儲存臉部特徵，和人臉辨識完全不同。
