16.1 排程提醒——HEARTBEAT.md

Heartbeat（心跳）是 OpenClaw 的「定時巡邏」機制。龍蝦會像盡責的管家一樣，每隔一段時間自動去做你交代的事，不需要你手動觸發。

HEARTBEAT.md 格式

檔案位置：~/.openclaw/workspace/HEARTBEAT.md（Windows 為 C:\Users\你的名稱\.openclaw\workspace\HEARTBEAT.md），用自然語言寫就好：

# HEARTBEAT.md

## 每日任務
- 每天早上 08:00，查詢台北天氣，用簡短的話告訴我穿衣建議
- 每天早上 08:30，查看 Google Calendar 今日行程
- 每天晚上 21:00，問我今天過得怎麼樣

## 每週任務
- 每週一早上 09:00，整理這週的待辦事項清單
- 每週五下午 17:00，回顧這週完成了什麼，給我鼓勵

## 提醒事項
- 3/15 提醒我繳信用卡帳單
💡 龍蝦會定期「醒來」檢查 HEARTBEAT.md（每 15-30 分鐘），判斷是否有任務需要執行。有就做，沒有就安靜繼續睡（回覆 HEARTBEAT_OK）。
這一頁怎麼做：
先開 記事本 / VS Code，編輯 C:\Users\你的名稱\.openclaw\workspace\HEARTBEAT.md。
如果檔案還不存在，可以在 PowerShell / 終端機 輸入：notepad $env:USERPROFILE\.openclaw\workspace\HEARTBEAT.md
存檔後再回到 PowerShell / 終端機 輸入：openclaw gateway restart
⚠️ 第一次設 HEARTBEAT，先只寫 1 到 2 個任務。先確認會動，再慢慢加，不要一開始就塞滿整份清單。
第一次驗證怎麼做：
先不要寫太多任務，先寫一條像「每天晚上 21:00 問我今天過得怎麼樣」。
到時間後，看 LINE / Telegram 有沒有主動收到訊息。

Heartbeat vs Cron & 早安助理範例

HeartbeatCron
精確度大約（±15 分鐘）精確到分鐘
設定方式自然語言寫在 HEARTBEAT.md指令設定
適合批次檢查、不需精確時間精確時間提醒、一次性任務
API 消耗較少（批次處理）較多（每個獨立呼叫）

實用範例：早安助理

# 早安助理（每天 07:30-08:00 之間執行）

每天早上幫我做以下事情，整合成一則訊息傳給我：
1. 查詢台北今天的天氣（溫度、降雨機率、穿衣建議）
2. 查看 Google Calendar 今天有什麼行程
3. 如果今天有會議，提醒我準備什麼
4. 用一句簡短的話幫我開啟美好的一天
✅ 龍蝦回覆：「🌤️ 早安！今天 26°C 多雲。帶傘出門比較保險。📅 10:00 部門會議 / 14:00 客戶拜訪。💡 記得帶上週五整理的報告！」
這個範例要先具備什麼：
這不是只靠 HEARTBEAT.md 就會成功。
你還要先把 weather 和 google-calendar 相關設定都做好。
如果你還沒接好 Google Calendar，先把第 2 點拿掉，先做只有天氣版的早安助理。

16.2 天氣查詢

安裝和設定

npx clawhub@latest install weather
# 在 TOOLS.md 加上：
## Weather
- API Key: 你的_OpenWeatherMap_API_Key
- 預設城市: 台北
- 溫度單位: 攝氏
這一段分別在哪裡做：
第 1 行安裝指令在 PowerShell / 終端機。
下面的 ## Weather 內容，是貼到 C:\Users\你的名稱\.openclaw\workspace\TOOLS.md。
存檔後要再跑 openclaw gateway restart。

日常使用方式

你
今天天氣如何？
龍蝦
🌤️ 台北今天 26°C，多雲偶陣雨，降雨機率 40%。建議帶把折疊傘，穿薄長袖即可。
你
明天適合洗車嗎？
龍蝦
🚗 明天台北有 70% 機率下午會下雨，建議後天再洗比較划算！
💡 龍蝦不只回報數字——它會根據天氣資訊給你實用的建議，這就是 AI + 技能的威力。
🔑 API Key 怎麼拿？ 到 openweathermap.org/api 免費註冊，每分鐘 60 次不用錢。詳細申請步驟請見附錄 E：Skills 設定速查表。
✅ 實作順序：安裝 weather → 補 TOOLS.md → openclaw gateway restart → 回 LINE 測「台北今天天氣」。
做完後先測這兩句：
請告訴我台北今天的天氣
明天台北適合洗車嗎？
如果這兩句都能正常回，就代表天氣技能大致接好了。

16.4 Notion 整合

整合龍蝦之後，你可以直接在 LINE 上跟龍蝦說話來操作 Notion——不用打開 Notion App、不用開電腦。

設定步驟

  1. 到 notion.so/my-integrations 建立 Integration
  2. 取得 Internal Integration Token（ntn_ 開頭）
  3. 在 Notion 頁面加入 Integration 的 connections
  4. 在 TOOLS.md 寫入 Token 和資料庫 ID
  5. 重啟 Gateway，再回 LINE 測試新增與查詢
這 5 步分別在哪裡做：
第 1 到 3 步都在 瀏覽器 的 Notion 後台。
第 4 步在 記事本 / VS Code 編輯 C:\Users\你的名稱\.openclaw\workspace\TOOLS.md。
第 5 步先回 PowerShell / 終端機 輸入 openclaw gateway restart，再回 LINE 測試。
最容易卡住的是第 3 步：
你就算拿到了 Token，如果沒有把 Integration 加進該頁面或資料庫的 connections，龍蝦還是讀不到 Notion。
TOOLS.md 最簡單可先這樣寫：
## Notion
- Token: 你的 ntn_... Token
- Database ID: 你的資料庫 ID

日常使用

你（走在路上）
幫我在 Notion 的待辦清單加一筆「買牛奶」
龍蝦
✅ 已加入 Notion 待辦清單：買牛奶
你
我的 Notion 待辦清單有什麼？
龍蝦
📋 你的待辦清單（3 筆未完成）：
1. ❌ 明天交報告（截止：3/8）
2. ❌ 買牛奶
3. ❌ 回覆客戶 Email
要標記哪一筆為完成嗎？
⚠️ 如果連不上，先檢查三件事：Token 對不對、connections 有沒有加、資料庫 ID 有沒有貼錯。
第一句先這樣測：
幫我在 Notion 的待辦清單加一筆「測試 Notion 整合」
如果新增成功，再問：我的 Notion 待辦清單有什麼？

16.5 Google Calendar 整合

整合龍蝦之後，你可以用說的來管行程——加行程、查行程、改行程，在 LINE 上一句話搞定。

設定步驟

  1. 到 Google Cloud Console 建立專案
  2. 啟用 Google Calendar API
  3. 建立服務帳號，下載 JSON 金鑰
  4. 把服務帳號 Email 加為行事曆共用對象（可編輯）
  5. 在 TOOLS.md 寫入憑證路徑
  6. 重啟 Gateway，回 LINE 測試新增行程
這 6 步分別在哪裡做：
第 1 到 4 步都在 瀏覽器 的 Google Cloud Console / Google Calendar。
第 5 步在 記事本 / VS Code 編輯 TOOLS.md。
第 6 步先回 PowerShell / 終端機 輸入 openclaw gateway restart，再回 LINE 測。
TOOLS.md 最簡單可先這樣寫：
## Google Calendar
- 服務帳號金鑰路徑: C:\Users\你的名稱\.openclaw\workspace\google-credentials.json
- 主要行事曆: primary
第 3 步下載的 JSON 要放哪裡？
最簡單就是放到 C:\Users\你的名稱\.openclaw\workspace\google-credentials.json
然後在 TOOLS.md 裡填同一條路徑，不要自己記兩套位置。

日常使用

你
幫我加一筆行程，下週二下午三點跟小明開會，一個小時
龍蝦
✅ 已新增行程：
📅 3/11（二）15:00-16:00
📝 跟小明開會
要設定提前 30 分鐘提醒嗎？
你
要
龍蝦
✅ 已設定提醒，會在 14:30 通知你。
⚠️ 如果 Google Calendar 沒反應，通常是三個地方有漏：Calendar API 沒啟用、服務帳號沒加入共用、JSON 路徑寫錯。
第一句先這樣測：
幫我加一筆行程，明天下午三點測試 Google Calendar，一個小時
如果成功，再去 Google Calendar 網頁確認這筆行程真的出現。

16.10 常見問題

問題解決方式
HEARTBEAT.md 任務沒有執行① HEARTBEAT.md 沒放在 C:\Users\你的名稱\.openclaw\workspace\
② 改完後忘了 openclaw gateway restart
③ Gateway 沒在跑 → 需持續運行
④ 時間未到（±15 分鐘誤差）
天氣技能回覆城市不對在 TOOLS.md 設定預設城市，或每次明確說「台北天氣」
Notion 連不上① API Token 是否正確（ntn_ 開頭）
② Integration 有沒有加入頁面 connections
③ 資料庫 ID 是否正確
Google Calendar 沒有反應① JSON 金鑰路徑錯誤
② 服務帳號沒有被加為共用對象
③ Calendar API 沒有啟用
這些技能會消耗很多 API 費用嗎？大部分都有免費額度：
OpenWeatherMap 每分鐘 60 次免費
NewsAPI 每天 100 次免費
Notion API 免費
Google Calendar API 每天 100 萬次
這章最穩的排查順序：
先看技能有沒有安裝。
再看 TOOLS.md / Token / JSON 路徑有沒有真的填對。
再跑 openclaw gateway restart。
最後才回 LINE / Telegram 重測。
