23.3 一行程式碼就能體驗

Page Agent 最厲害的地方——在任何網頁加一行 <script>，就能用自然語言操控它。

方法一：在任何 HTML 加一行

<script src="https://cdn.jsdelivr.net/npm/page-agent@1.5.7/dist/iife/page-agent.demo.js"
        crossorigin="true"></script>
⚠️ 這行使用 Page Agent 的免費測試 LLM，不需要 API Key。適合學習體驗用，不適合正式環境。

方法二：npm 安裝，指定自己的 LLM

npm install page-agent
import { PageAgent } from 'page-agent'

const agent = new PageAgent({
    model: 'qwen3.5-plus',              // 或 gpt-4o / claude-3.5-sonnet
    baseURL: 'https://你的API位址/v1',
    apiKey: '你的API_KEY',
    language: 'zh-TW',
})

await agent.execute('在搜尋框輸入龍蝦食譜，然後按搜尋')
✅ 支援的 LLM：Qwen、GPT、Claude、Ollama（本地）——和龍蝦用的是同一批 AI 模型！

23.4 核心程式碼（20 行）

這就是龍蝦操控網頁的完整程式碼——比你想的簡單很多。

// web-control.js — 龍蝦的網頁操控腳本
const { chromium } = require('playwright');

async function controlWebPage(url, task) {
    // 1. 開啟瀏覽器（headless: false 讓你看到操作過程）
    const browser = await chromium.launch({ headless: false });
    const page = await browser.newPage();

    // 2. 打開目標網頁
    await page.goto(url, { waitUntil: 'networkidle' });

    // 3. 注入 Page Agent SDK（免費測試版）
    await page.addScriptTag({
        url: 'https://cdn.jsdelivr.net/npm/page-agent@1.5.7/dist/iife/page-agent.demo.js'
    });

    // 4. 等 Page Agent 載入，然後執行任務
    await page.waitForTimeout(2000);
    const result = await page.evaluate(async (task) => {
        const agent = new PageAgent({ language: 'zh-TW' });
        return await agent.execute(task);
    }, task);

    // 5. 完成，關閉瀏覽器
    await browser.close();
    return result;
}

// 使用範例
controlWebPage('https://google.com', '搜尋龍蝦食譜');
💡 headless: false 表示打開可見的瀏覽器視窗，學生能看到 AI 操作過程。正式使用可改為 true（背景執行）。

免費測試 LLM 適合學習，但有使用限制。正式使用建議換成自己的 API。

支援的 LLM 和設定方式

LLMmodel 參數baseURL

OpenAIgpt-4ohttps://api.openai.com/v1
Anthropicclaude-3.5-sonnethttps://api.anthropic.com/v1
Ollama（本地免費）qwen2.5:7bhttp://localhost:11434/v1

修改 web-control.js

const agent = new PageAgent({
    model: 'qwen3.5-plus',                         // 改這裡
    baseURL: 'https://dashscope.aliyuncs.com/compatible-mode/v1',  // 改這裡
    apiKey: process.env.QWEN_API_KEY,              // 從環境變數讀取

23.7 常見問題

  • Q1：Page Agent 能處理驗證碼嗎？
    A：不能。驗證碼（CAPTCHA）的目的就是擋機器人。遇到驗證碼時 Page Agent 會卡住。建議用在不需要登入的公開網站。
  • Q2：能登入需要帳號密碼的網站嗎？
    A：技術上可以，但不建議把密碼寫在腳本裡。如果需要登入，建議先手動登入保持 session，再讓 Page Agent 操作。
  • Q3：速度快嗎？
    A：每個動作大約 2-5 秒（因為要呼叫 LLM 判斷）。填一張 5 欄的表單約 15-30 秒。不如手動快，但勝在不需要人盯著。
  • Q4：會不會不小心點錯東西？
    A：Page Agent 有 human-in-the-loop 機制——每一步都可以設定要不要人工確認。高風險操作（如付款、刪除）建議開啟確認。
  • Q5：免費嗎？
    A：Page Agent 本身是 MIT 開源免費。免費測試 LLM 有使用限制，正式使用需要自己的 LLM API（可以用龍蝦已有的）。
