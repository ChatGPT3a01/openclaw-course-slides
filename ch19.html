19.1 Nodes 多機架構介紹

你的龍蝦現在只住在「一台電腦」上。但如果你有好幾台電腦呢？Nodes 功能讓你把多個節點串在一起，形成一個「龍蝦網路」。

為什麼需要多節點？

情境單機的限制多節點的解法
電腦關機就斷線龍蝦下線，無法接收訊息雲端節點 24hr 接收，本機負責重工作
多台電腦要管理每台各自裝龍蝦，互不相干多節點串聯，統一管理
運算效能不夠小筆電跑 AI 很慢把 AI 運算丟給效能好的節點
多人協作每人各自操作龍蝦之間可以互相傳遞任務
2 / 10

節點的角色：Primary vs Worker

👔 Primary（主控節點）

比喻：公司的總經理

  • 接收所有通道的訊息（LINE、Telegram 等）
  • 判斷任務要交給誰
  • 彙整結果回覆使用者
  • 維持節點清單
通常放在雲端伺服器或 24 小時開機的電腦上

🔧 Worker（工作節點）

比喻：各部門的員工

  • 向主控註冊「我上線了」
  • 接收主控分派的任務
  • 執行任務（跑 AI、操作檔案等）
  • 完成後回報結果
可以是家用電腦、辦公室筆電、或另一台雲端伺服器 
        ┌─────────────┐
        │   LINE 訊息   │
        └──────┬──────┘
               │
        ┌──────▼──────┐
        │   Primary    │  ← 主控節點（雲端伺服器）
        │  接收訊息     │     24 小時在線
        │  分派任務     │
        └──┬───────┬──┘
           │       │
    ┌──────▼──┐ ┌──▼──────┐
    │ Worker A │ │ Worker B │
    │ 家裡桌機  │ │ 辦公室筆電│
    │ 跑 AI 運算│ │ 操作檔案  │
    └─────────┘ └─────────┘
3 / 10

節點能力標籤（NODE.md）

每個工作節點可以宣告自己的「能力」，讓主控知道該把什麼任務分派給它：

# NODE.md（工作節點的設定檔）

## 節點名稱
家裡桌機

## 能力
- desktop-agent（可操控瀏覽器和檔案）
- gpu-compute（有好顯卡，可跑本地 AI）
- file-access（可存取家裡的 NAS）

## 在線時段
- 平日：18:00-23:00
- 假日：全天

主控節點的分派邏輯

🖥️ 操控桌面

分給「家裡桌機」（有 desktop-agent 能力）

🧠 跑本地 AI

分給「家裡桌機」（有 GPU）

🌙 凌晨任務

不分給「家裡桌機」（可能關機了），改由雲端處理

💡 一個 Nodes 網路裡只有一個 Primary，但可以有多個 Worker（最多 5 個）。
4 / 10

19.3 設定多台龍蝦分工合作

步驟一：設定主控節點

# 啟用 Nodes 功能
openclaw config set nodes.enabled true

# 設定自己為 Primary
openclaw config set nodes.role primary

# 產生節點金鑰（讓其他節點能安全連線）
openclaw nodes init
🔑 Nodes 初始化完成！
   節點 ID：node-abc123
   連線金鑰：sk-nodes-xxxxxxxxxxxxxxxx

   ⚠️ 請把連線金鑰複製給你的工作節點。
   這個金鑰只顯示一次，請妥善保管。

步驟二：設定工作節點

# 啟用 Nodes 功能
openclaw config set nodes.enabled true

# 設定自己為 Worker
openclaw config set nodes.role worker

# 連線到主控節點
openclaw nodes join --primary https://你的主控域名:18789 --key sk-nodes-xxxxxxxxxxxxxxxx
✅ 已成功連線到主控節點！
   主控：https://lobster.你的域名.com:18789
   我的節點 ID：node-def456
   我的角色：Worker
   延遲：32ms
5 / 10

19.5 注意事項與常見問題

網路與安全

項目說明
連線加密節點之間的通訊使用 TLS 加密，不用擔心被竊聽
金鑰管理連線金鑰只在 nodes join 時使用一次，之後用節點 ID 認證
防火牆工作節點不需要開放 Port，它是主動連出去的
斷線重連工作節點斷線後會自動嘗試重連（間隔 30 秒）

目前的限制

📊 最多 5 個 Worker

一個 Nodes 網路最多支援 5 個工作節點

📦 檔案大小限制

跨節點的檔案傳輸預設上限 100MB

↩️ 回覆路徑

工作節點不能直接回覆使用者，必須透過主控節點

🔄 分派邏輯

任務分派邏輯還比較基礎，不像企業級任務佇列

💡 Nodes vs CH14 多重分身：多重分身是「一台電腦上跑多個 Gateway」（多個獨立龍蝦）；Nodes 是「多台電腦上的龍蝦串在一起」（一個龍蝦網路）。兩者可搭配使用。
9 / 10

19.7 小結與展望

🌐 Nodes 架構

Primary 主控節點 + 多個 Worker 工作節點，形成「龍蝦網路」。

⚙️ 節點設定

openclaw nodes init 初始化主控 + openclaw nodes join 加入工作節點。

📋 能力宣告

NODE.md 定義每個節點的能力和在線時段，主控按能力分派任務。

🏠 實戰範例

家庭+雲端、辦公室+家裡、多人協作——讓龍蝦不再只住在一台電腦裡。
