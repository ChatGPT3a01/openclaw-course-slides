15.1 什麼是 Skills？

Skills 就像手機上的 App——你的手機本身可以打電話、拍照，但你裝了 Uber 才能叫車、裝了 Foodpanda 才能訂餐。龍蝦也一樣，裝了天氣技能才能查天氣、裝了行事曆技能才能管行程。

在 OpenClaw 的架構裡，Skill 是一個擴充龍蝦能力的模組。每個 Skill 定義了：

  • 龍蝦能做什麼新的事（例如查天氣、寄 Email）
  • 怎麼跟外部服務互動（例如呼叫天氣 API、連接 Gmail）
  • 什麼時候觸發（例如使用者說「天氣」的時候）

第一次使用 Skills，先照這個順序

  1. 先開 PowerShell / 終端機，輸入：npx clawhub@latest install weather
  2. 把需要的設定補進 TOOLS.md：Windows 路徑是 C:\Users\user\.openclaw\workspace\TOOLS.md
  3. 重啟 Gateway：openclaw gateway restart
  4. 回到 LINE，用最明確的一句話測試，例如：請告訴我台北今天的天氣
這 4 步分別在哪裡做：
第 1、3 步在 PowerShell / 終端機。
第 2 步在 記事本或 VS Code 編輯 TOOLS.md。
第 4 步回到 LINE / Telegram 跟龍蝦實測。
如果還沒有 TOOLS.md，就直接建立：
在 PowerShell 輸入 notepad $env:USERPROFILE\.openclaw\workspace\TOOLS.md
如果檔案不存在，記事本會自動幫你新建。
天氣技能最簡單怎麼補？
把下面這段直接貼進 TOOLS.md：

## Weather（天氣）
- 預設城市: 台北
- 溫度單位: 攝氏

如果這個技能版本需要 API Key，再補：
- API Key: 你的 OpenWeatherMap API Key
為什麼要用「最明確的一句話」測試？
因為你如果只打「天氣」兩個字，龍蝦可能不知道你要查哪個城市、今天還是明天。
像 請告訴我台北今天的天氣 這種句子，技能比較容易被正確觸發。
這一步做完後，你至少要確認 3 件事：
openclaw skills list 看得到 weather。
TOOLS.md 已經存在，而且內容已存檔。
回 LINE 問天氣時，龍蝦不是亂聊，而是真的回天氣內容。
💡 一句話總結：AI 模型本身「會思考」，Skills 則讓龍蝦「能行動」。

15.1.1 什麼是 Tools？和 Skills 有什麼差別？

🧰 Tools

意思：龍蝦的能力、權限、可用操作。

重點：決定它能不能做某件事。

例子：讀檔、改檔、執行命令、搜尋網路、操作瀏覽器。

📚 Skills

意思：做事的方法、任務規則、場景知識。

重點：教龍蝦怎麼把 Tools 組合起來完成任務。

例子：weather、gog、summarize。

最好記的版本：
Tools = 能力開關
Skills = 任務教學
很重要：裝了 Skill，不代表龍蝦就自動多了新權限。
Skill 比較像說明書。真正能不能做到，還要看對應的 Tool / 外掛 / 設定 / 授權有沒有準備好。
例如你裝了某個需要讀檔、上網或連外服務的 Skill，但相關能力沒開、Token 沒貼、帳號沒授權，它還是做不了。
用天氣技能當例子：
Tool 層：要有查資料、讀設定、回傳訊息的能力。
Skill 層：告訴龍蝦什麼時候該查天氣、去哪裡查、怎麼整理成「今天要不要帶傘」這種回覆。
這一章你現在主要在學的是：
安裝 Skill、補設定、重啟、測試。
也就是先把「任務方法」裝好，再確保相關設定真的可用。
補充一個觀念：
Skills 不是只有 OpenClaw 才有。
Anthropic / Claude 也有相同思路的技能設計。
所以你現在學的，不只是某個產品功能，而是一種「把 AI 能力包成可重複使用模組」的方法。
相關參考可以在 CH17 看到 anthropics/skills 的連結。

Skill 運作流程

當你跟龍蝦說「今天台北天氣如何？」，背後發生了六個步驟：

① 傳訊息

你：「今天台北天氣如何？」

② AI 判斷

龍蝦的 AI 大腦判斷：這是在問天氣

③ 選技能

AI 決定使用「天氣技能」

④ 呼叫 API

天氣技能去呼叫天氣 API，拿到資料

⑤ 整理回覆

AI 把天氣資料整理成自然語言

⑥ 回覆你

「今天台北 26°C，多雲偶陣雨...」

✅ 關鍵：AI 模型會自己判斷該用哪個技能。你不需要打特殊指令，用自然語言說話就好。
⚠️ 技能裝了卻沒反應，最常見就是少了兩件事：TOOLS.md 沒補設定，或忘記 openclaw gateway restart。
這一頁最該記的是：
你平常還是跟龍蝦自然聊天。
安裝技能 是在 PowerShell。
觸發技能 是回 LINE / Telegram 跟龍蝦講自然語言。

SKILL.md 格式

每個技能都有一個 SKILL.md 定義檔，告訴龍蝦這個技能是什麼、能做什麼：

---
name: weather
description: 查詢全球各城市的即時天氣、未來預報和氣象資訊。
---

# Weather（天氣查詢）

目標：根據使用者提供的城市或位置，查詢即時天氣資訊。

## 功能
- 查詢即時天氣（溫度、濕度、風速、天氣狀況）
- 查詢未來 3-7 天預報
- 紫外線指數 / 穿衣建議

## 使用方式
- 「今天天氣怎麼樣？」
- 「台北明天會下雨嗎？」
- 「東京這週的天氣預報」

## 需要的設定
- API Key：在 TOOLS.md 裡設定 OpenWeatherMap API Key
- 預設城市：在 TOOLS.md 裡設定使用者所在城市
💡 龍蝦讀了這個檔案，就知道這個技能的用途和使用時機——就像招聘時的「職位說明書」。
學生先不用自己寫 SKILL.md。
這一頁是讓你知道技能裡面長什麼樣。
這一章先學會 安裝技能、補設定、重啟、測試 就夠了。
自己寫技能會在後面章節再教。

15.3 Skills 安裝與管理

搜尋技能

npx clawhub@latest search weather

安裝技能

npx clawhub@latest install weather
# 或
openclaw skills install weather

查看已安裝技能

openclaw skills list
這一整頁的指令都在 PowerShell / 終端機 輸入。
不是貼到 LINE、Telegram，也不是貼到 TOOLS.md 裡。

🔄 啟用 / 停用

openclaw skills disable weather
openclaw skills enable weather

⬆️ 更新

npx clawhub@latest update weather
npx clawhub@latest update --all

🗑️ 移除

openclaw skills remove weather
⚡ 每次安裝、移除或修改技能後，都要重啟 Gateway：openclaw gateway restart

安裝完馬上這樣檢查

  1. 輸入 openclaw skills list，確認技能真的有出現在清單裡
  2. 如果需要 API Key，就立刻補進 TOOLS.md
  3. 執行 openclaw gateway restart
  4. 回 LINE 測一句最具體的指令，例如「查台北天氣」
如果第 1 步清單裡根本看不到技能：
先不要急著回 LINE 測。
代表安裝根本還沒成功，先回頭重新執行安裝指令。

1tavily-search生產力聯網搜尋，沒這個等於瞎子
2weather生活實用最實用日常技能，免 API Key
3find-skillsAgent 智慧讓龍蝦自己找工具安裝
4summarize生產力長文/PDF/郵件一鍵抓重點
5agent-browser通訊語音操控瀏覽器，填表單查資料
6gog生產力Google 全家桶一站搞定
7self-improving-agentAgent 智慧越用越聰明，記住你的偏好
8proactive-agentAgent 智慧從被動執行變主動規劃
npx clawhub@latest install skill-vetter tavily-search weather find-skills summarize agent-browser gog self-improving-agent proactive-agent
這串指令是什麼意思？
npx clawhub@latest install 是「用最新版本的 ClawHub 安裝技能」。
後面那 9 個英文名字，就是要一次安裝的 9 個技能名稱。
也就是說，這一整行的意思其實是：一次把這 9 個技能全部裝進龍蝦。
不一定要一次全裝。
如果你是第一次上手，建議先裝 1 到 3 個最需要的就好。
例如先裝：weather、tavily-search、summarize
寫法是：npx clawhub@latest install weather tavily-search summarize
這 9 個名字不是說明文字，是技能代號：
skill-vetter 安全審查技能、tavily-search 聯網搜尋、weather 天氣、find-skills 找技能、summarize 摘要、agent-browser 瀏覽器操控、gog Google 工具整合、self-improving-agent 自我優化、proactive-agent 主動規劃。
這一行指令要在哪裡打？
在你平常安裝 OpenClaw 的 PowerShell / 終端機 裡打。
打完後再輸入 openclaw skills list 檢查有沒有真的裝進去。

15.5 Skills 設定與客製化

很多技能需要額外設定才能使用——最常見的就是 API Key。

方式一：寫在 TOOLS.md 裡

## Weather（天氣）
- API Key: 你的OpenWeatherMap_API_Key
- 預設城市: 台北
- 溫度單位: 攝氏

## Google Calendar
- 服務帳號金鑰路徑: ~/.openclaw/workspace/google-credentials.json
- 主要行事曆: 工作

方式二：用 CLI 設定

openclaw config set skills.weather.apiKey 你的API_Key
openclaw config set skills.weather.defaultCity 台北
這兩種做法分別在哪裡做：
方式一：用 記事本 / VS Code 編輯 C:\Users\user\.openclaw\workspace\TOOLS.md。
方式二：在 PowerShell / 終端機 輸入 openclaw config set ...。
新手先用哪一種？
建議先用 方式一：寫在 TOOLS.md。
因為你看得到、改得到，也比較好教學。
CLI 設定比較適合熟手或自動化。

客製化技能行為

你可以修改技能的 SKILL.md 來客製化行為——例如改成「回覆天氣時用簡短一句話就好」，龍蝦就會照辦。

這一步先知道就好：
新手目前先不要急著改技能資料夾裡的 SKILL.md。
先把技能裝好、設定好、可以正常使用，比較重要。
💡 技能的檔案結構：每個技能是 ~/.openclaw/workspace/skills/ 裡的一個獨立資料夾，包含 SKILL.md 和（可選的）腳本。
📖 所有技能的 API Key 申請步驟和 TOOLS.md 設定範例，請參考附錄 E：Skills 設定速查表——每個技能都有手把手的設定指南。
這一頁做完後，最穩的驗證方式：
存好 TOOLS.md 或輸入完 openclaw config set ... 後，執行 openclaw gateway restart。
然後回到 LINE / Telegram，用該技能最明確的句子測一次。

15.6 常見問題

問題解決方式
安裝了技能但龍蝦不會用① 忘記重啟 → openclaw gateway restart
② API Key 沒設定 → 檢查 TOOLS.md
③ 技能被停用 → openclaw skills list 確認
④ 說法太含糊 → 明確說「用天氣技能查台北天氣」
技能太多龍蝦搞混控制在 10 個以內，功能重複的只留一個
我可以自己寫技能嗎？可以！CH17 會專門教你，門檻不高
技能需要花錢嗎？技能本身免費，但部分 API 有費用（大部分有免費額度）
技能可以跨通道嗎？可以。技能跟龍蝦綁定，不跟通道綁定。LINE / TG / Discord 都用同一個技能
哪裡找更多技能？npx clawhub@latest search 關鍵字、GitHub、OpenClaw Discord 社群
學生最常卡住的排查順序：
先看 openclaw skills list 裡有沒有安裝成功。
再看 TOOLS.md 或 API Key 有沒有真的存好。
再跑 openclaw gateway restart。
最後才回 LINE / Telegram 重測。
