6.1 第一場真正對話 & 五種對話模式

打開 LINE，找到你的龍蝦 Bot，傳一則訊息：你好，我是你的主人，請自我介紹一下。龍蝦的回覆風格取決於你的 IDENTITY.md 人格設定。

開始這一章前，先確認兩件事：
1. 你在 CH5 已經完成配對，LINE 傳訊息時龍蝦真的會回。
2. 如果你想看到自己設定的人格語氣，IDENTITY.md 要放在 C:\Users\user\.openclaw\workspace\IDENTITY.md，改完後要執行 openclaw gateway restart。

龍蝦會記住你們聊過的內容——這就是多輪對話的基礎。接下來認識五種對話模式：

💬 閒聊模式

像跟朋友傳訊息一樣隨意聊天。

「推薦我一部好看的電影」

📚 問答模式

把龍蝦當百科全書，問清楚就能得到精準答案。

「解釋區塊鏈，用小學生聽得懂的方式」

🎯 任務模式

請龍蝦幫你做一件具體的事：翻譯、寫文、列表。

「幫我列一份三天兩夜台南行程表」

🎭 角色扮演

讓龍蝦扮演特定角色與你互動。

「你是英文老師，我說中文你幫我翻」

🔄 多輪修改

先產出初稿，再一步步修改到滿意。

「太長了，縮短到 100 字以內」

6.2-6.3 對話模式詳解 & 四大對話技巧

問答模式：問法決定回覆品質

模糊的問法精準的問法
推薦餐廳推薦台北信義區適合四人聚餐的日式餐廳，預算每人 800 元
幫我寫信幫我寫一封請假信給主管，下週三要看牙醫，語氣要正式禮貌
這個怎麼用Node.js 的 fs.readFile 要怎麼用？請給我一個讀取 JSON 檔的範例

四大對話技巧——讓龍蝦更懂你

1️⃣ 提供背景資訊

「我是國小老師，要教五年級學生認識分數。幫我設計三個有趣的課堂活動。」

加上身份和情境，龍蝦就不會給你大學程度的教材。

2️⃣ 指定輸出格式

「用表格列出五種程式語言的比較，欄位：名稱、適合用途、學習難度」

表格、條列、Markdown——直接說你要什麼格式。

3️⃣ 給範例

「範例：王小明 — 經理。請把以下名單也整理成同樣格式。」

文字描述不清楚時，給一個範例最有效。

4️⃣ 分步驟拆解

「第一步：先幫我列大綱。」等龍蝦回覆後再說：「第二步：展開第三點。」

複雜需求拆成小步驟，比一次丟完好得多。

6.4 實戰對話範例——跟著做就對了

三個範例涵蓋不同場景，建議你打開 LINE 實際操作。每個範例都遵循四輪對話的模式。

📝 學生寫報告

第一輪：給背景，請龍蝦寫 800 字心得報告初稿

第二輪：加入自己的真實例子

第三輪：修改結尾語氣

第四輪：加上標題、學號、日期格式

📋 會議紀錄整理

第一輪：丟雜亂筆記，請龍蝦整理成正式紀錄

第二輪：指定表格格式（決議事項、負責人、截止日）

第三輪：順便寫一封寄送 Email

從筆記到正式紀錄，不到三分鐘！

🧮 老師準備考題

第一輪：出 10 題一元一次方程式（難度由易到難）

第二輪：調整第 7 題為生活化情境

第三輪：分成學生版和教師版

第四輪：加計分表和姓名欄位

✅ 四步模式：① 給背景 → ② 先得到粗略版本 → ③ 逐步修改 → ④ 最後處理格式。記住這個模式，對話品質會好很多！
學生可以直接照這個順序練一次：
第一則先下完整需求。
第二則只改一件事，例如「改正式一點」或「縮成 200 字」。
第三則再補格式，例如「改成條列式」或「加標題」。
不要一開始就丟十個要求，龍蝦比較容易抓錯重點。

6.5 LINE 聊天室的操作須知

在 LINE 上跟龍蝦互動，有一些平台特有的操作和限制需要知道。

✏️ 文字訊息

LINE 文字訊息沒有字數上限（技術上最大 5000 字元），可以放心貼長文。

龍蝦的回覆如果超過 5000 字元，會自動分成多則訊息傳送。

😊 貼圖（Sticker）

你可以傳 LINE 貼圖給龍蝦，但它看不太懂貼圖的含義。

想表達心情，用文字加上 emoji 比較有效。

🖼️ 圖片訊息

如果 AI 模型支援視覺功能（Gemini / GPT-5.2 / Claude），龍蝦就能看懂照片。

→ 詳見 CH8 拍照與視覺辨識

🎤 語音訊息

LINE 語音訊息目前 OpenClaw 尚未支援。龍蝦會告訴你它暫時無法處理語音。

→ 語音通話功能見 CH20

✓ 訊息已讀

龍蝦收到訊息後立刻標記已讀，然後開始處理。

回覆速度取決於模型：Gemini Flash 約 1-2 秒，Claude Opus 約 5-10 秒。

6.6 龍蝦的記憶力

記憶功能是讓龍蝦從「一問一答的工具」升級為「持續對話的夥伴」的關鍵。

🧠 短期記憶——對話上下文

在同一段對話中，龍蝦記住你們聊過的所有內容。

你：推薦三家火鍋店
龍蝦：（列出三家）

你：第二家評價如何？
龍蝦：（不需重複店名，直接回答）

你：那它交通方便嗎？
龍蝦：（知道「它」是第二家）

💾 長期記憶——跨對話記憶

龍蝦會把重要資訊存進長期記憶，隔天還記得。

（今天）
你：記住，我對海鮮過敏

（明天）
你：推薦我今天的午餐
龍蝦：考慮到你對海鮮過敏，
我推薦以下幾間...

長期記憶存放在 workspace/ 資料夾中。

🗑️ 清除對話

想重新開始全新對話（清除短期記憶），有三種方式：

  • LINE 上傳送 /clear
  • CLI：openclaw conversations clear
  • Dashboard 對話紀錄頁面刪除

長期記憶需手動到 workspace/ 編輯。

💡 小技巧：對話太長時，較早的訊息會被自動裁剪（短期記憶有容量限制）。重要的資訊讓龍蝦存進長期記憶比較保險：「記住這件事：......」
這頁要先講清楚一件事：
這裡說的 workspace/，Windows 最常見路徑就是 C:\Users\user\.openclaw\workspace\。
IDENTITY.md、SOUL.md、USER.md 都是在這個資料夾裡。
改完這些檔案後，記得重啟一次：openclaw gateway restart。

6.8 LINE 平台的限制與注意事項

📊 免費方案訊息額度

每月 200 則推播訊息（Push Message）限制。

但你傳訊息後龍蝦的回覆（Reply）不計次數、免費！

日常對話完全不用擔心額度問題。只有 CH21 的「排程推播」才需注意。

🚫 自動回覆必須關閉

如果龍蝦突然回「感謝您的訊息，我們將盡快回覆」之類的罐頭訊息——那是 LINE 內建的自動回覆被開啟了。

回到 LINE Official Account Manager，確認「自動回應訊息」是停用狀態。

👥 好友人數

免費方案最多 500 位好友。

對個人使用和教學來說綽綽有餘。

📱 Rich Menu（圖文選單）

很多 LINE 官方帳號底部都有一排按鈕選單，那就是 Rich Menu。龍蝦也能設定，讓常用功能一鍵觸發。

→ 詳見 CH11 LINE 進階設定

⚠️ 重要提醒：免費方案的 200 則推播額度是指龍蝦「主動」傳給你的訊息。你傳訊息給龍蝦後龍蝦的「回覆」是免費的，完全不計次數。

6.9 取得你的 LINE User ID

後面好幾個章節都會用到 userId——推播訊息、排程通知、語音通話都需要它。先搞清楚它是什麼、怎麼拿到手。

如果你前面章節已經看過 userId 這個詞，詳細怎麼取得就在這裡：
本章的 6.10 到 6.13 會把三種取得方式一步一步講完。

💬 回覆（Reply）

使用者先說話 → 龍蝦回覆

不需要知道 userId，用 replyToken 就能回

像是「有人問你，你回答」——免費、不限次數

📣 推播（Push）

龍蝦主動傳訊息（使用者沒先說話）

必須知道 userId 或 groupId

像是「你主動傳訊息給朋友」——每月 200 則免費

⚠️ userId 不是 LINE ID！
userId 是系統產生的 U 開頭 33 字元字串（例如 U1a2b3c4d5e6f7g8h9i0j...），跟你設定的 LINE ID（@xxx）完全不同。無法自己查，只能透過程式取得。
這一章真正要你做的事只有一件：
把自己的 userId 找出來，然後存進記事本。
建議直接記在 龍蝦課程-帳號清單.txt，後面 CH11、CH20、CH21 都會再用到。

6.10 三種方法取得 User ID

方法一：Dashboard 查看（推薦）

1. 先傳一則訊息給龍蝦（讓系統記錄你的資訊）

2. 打開 Dashboard
http://localhost:18789

3. 點「對話紀錄」頁面

4. 找到你的對話，userId 顯示在使用者資訊中

5. 複製 U 開頭字串，存到記事本

最簡單！不需要額外工具

方法二：Gateway 終端機日誌

1. 在終端機啟動 Gateway

openclaw gateway start

2. 用手機 LINE 傳一則訊息

3. 終端機會顯示收到的訊息資訊，包含 userId

4. 複製 U 開頭字串

適合進階使用者和除錯

方法三：GAS 查詢小工具

原理：建一個 Google Apps Script 小 Bot，傳訊息就自動回覆你的 userId

優點：不需要 OpenClaw，完全獨立運作

適用：還沒安裝 OpenClaw 的人，或需要取得群組 groupId

步驟：建立 GAS → 貼程式碼 → 填 Token → 部署 Web App → 設 Webhook → 傳訊息取得 ID

程式碼在本章上方的「⬇️ GAS 程式碼」，完整排錯請看附錄 B 疑難排解大全

🎯 取得 userId 後，把它存在記事本裡——只要對方沒有封鎖你的 Bot，userId 就永遠有效，不需要重複取得。之後在 HEARTBEAT.md、推播指令、技能設定中直接貼上即可。
新手建議優先順序：
已經裝好 OpenClaw 的人，先用 方法一 Dashboard。
如果你正在除錯，才用 方法二 終端機日誌。
如果你還沒裝 OpenClaw，或想另外查群組 ID，再用 方法三 GAS。

取得群組 groupId

如果要推播到群組：把龍蝦邀請加入群組 → 在群組中傳一則訊息 → Dashboard 或終端機日誌中就會出現 groupId（C 開頭字串）。userId 推播給個人，groupId 推播到群組。

6.11 GAS 手把手教學（上）— 建立查詢 Bot

用 Google Apps Script 建一個「User ID 查詢 Bot」——傳訊息就自動回覆你的 userId。不需要 OpenClaw，完全獨立運作！

Step 1：開啟 Google Apps Script

👆 動作：打開瀏覽器，前往 script.google.com
👆 動作：點擊左上角「新專案」
👀 結果：打開 GAS 編輯器，裡面有一個空的 myFunction()

Step 2：改專案名稱

📍 位置：左上角「未命名的專案」
👆 動作：點擊它，改名為「取得 User ID」

Step 3：貼上程式碼

👆 動作：把編輯器裡原本的 myFunction() 全部刪除
👆 動作：點簡報上方的「⬇️ GAS 程式碼」下載 .gs 檔案
👆 動作：用記事本打開 → Ctrl+A 全選 → Ctrl+C 複製 → 回到 GAS 編輯器 Ctrl+V 貼上
👀 結果：編輯器裡出現完整的程式碼

Step 4：填入 Channel Access Token

📍 位置：程式碼第 17 行
👀 你會看到： 
var LINE_TOKEN = '在此貼上你的 Channel Access Token';
👆 動作：到 LINE Developers Console → 你的 Bot → Messaging API → 複製 Channel access token
👆 動作：把引號裡的中文替換成你的 Token → 按 Ctrl+S 儲存
👀 結果：看起來像 'xYz123abc456def789ghi...'（Token 很長，正常的）
這裡要貼的是哪一個 Token？
要貼的是 Channel Access Token，不是 Channel ID，也不是 Channel Secret。
如果貼錯，後面 Bot 不會正常回你 userId。

👀 結果：部署成功後，會顯示一個網址
👆 動作：點「複製」把網址複製下來 
https://script.google.com/macros/s/AKfycb.../exec

Step 7：設定 LINE Bot Webhook

⚠️ 重要！如果 Webhook URL 已經有設定網址（例如 ngrok 或 Cloudflare），先複製到記事本備份！等一下取得 ID 後要改回來。
👆 動作：到 LINE Developers Console → 你的 Bot → Messaging API
👆 動作：找到 Webhook settings → 點「Edit」

所以原本 OpenClaw 用的 Webhook 網址一定要先備份，等拿到 userId 之後再改回來。

⚠️ 最重要！取得 userId 後，立刻把 Webhook URL 改回原本的網址（你備份的那個）。否則龍蝦只會一直回覆 ID，其他功能全部失靈！
🎯 userId 永不過期——只要對方沒封鎖 Bot，userId 就永遠有效。存好一次，之後在 HEARTBEAT.md、CLI 推播、技能設定中直接貼上即可。
改回去時怎麼改？
回到 LINE Developers Console 的 Messaging API → Webhook settings。
把 GAS 網址刪掉，貼回你剛才備份的原始網址，例如 https://xxxx.ngrok-free.app/line/webhook。
按 Save 後，再按一次 Verify 確認恢復成功。

問題解決方法
龍蝦回覆很慢，要等很久 回覆速度取決於 AI 模型。Gemini Flash 最快（1-2 秒），Ollama 本地模型可能 5-15 秒。超過 30 秒沒回覆，先確認 Gateway 還在運行：openclaw gateway status
龍蝦回覆被截斷，說到一半 回覆超過 AI 模型的輸出長度限制。在問題中加上「請簡短回覆」或「控制在 500 字以內」。
龍蝦突然不認識我了，好像失憶 短期記憶有容量限制，對話太長時較早的訊息會被裁剪。重要資訊讓龍蝦存進長期記憶：「記住這件事：...」
